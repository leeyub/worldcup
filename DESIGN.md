# DESIGN.md — SKT 35기 월드컵응원단 디자인 스펙 (초안)

> ⚠️ **이 문서는 "의도(intent) 문서"입니다.**
> 토큰 값의 **원본(source of truth)은 `index.html`의 CSS `:root`** 이고, 이 문서는 그 값들이 *왜 그렇게 정해졌는지, 어떻게 써야 하는지*를 설명합니다.
> 값이 서로 다르면 **코드(`:root`)가 정답**입니다. 코드를 바꾸면 이 문서도 같이 손봐 주세요.
> 작성: Claude (초안) · 다듬기: 이동엽(오너/UX 디자이너). 자유롭게 고쳐 쓰세요.
>
> 기본기(타입 스케일·스페이싱·elevation·상태)는 **Material Design 3**([m3.material.io](https://m3.material.io))의 파운데이션을 참고해 *우리 규모에 맞게* 축약·매핑했습니다. 전부 따르지 않고 초안 품질에 도움되는 부분만 차용했습니다.

## 1. Overview

회사 동기 8명이 2026 월드컵 한국 조별리그 3경기를 함께 보며 쓰는 **예측 게임 웹앱**. 각자 폰으로 접속해 결과를 예측하고, 어드민이 실시간으로 골·카드를 입력하면 점수와 순위가 자동으로 바뀝니다.

- **성격**: 친구용 비공식 게임. 가볍고, 즐겁고, 폰에서 빠르게.
- **제약**: `index.html` 단일 파일(HTML+CSS+JS), 프레임워크·빌드 없음. 외부 의존성은 supabase-js CDN 하나.
- **현재 테마**: 다크가 기본. 라이트는 토글(🌙/☀️)로 켜고, 선택은 `localStorage`에 저장.
- **디자인 톤**: 스포츠 응원의 에너지(레드/블루/골드) + 모바일 카드 UI의 정돈됨.
- **검수 원칙**: 모든 UI 변경은 `/demo` 모드(가짜 데이터 + 하단 패널)에서 역할 3종 × 경기상태 7종으로 확인.

**두 테마의 의도**
| | 다크 (기본·현황) | 라이트 (방향·의도) |
|---|---|---|
| 분위기 | 야간 경기 관람, 또렷한 대비 | "뽀얀"·밝고 깨끗 (Fixtured 앱 레퍼런스) |
| 배경 | 짙은 네이비 | 밝은 뉴트럴 그레이 + 흰 카드 |
| 깊이 | 테두리로 구분(플랫) | 소프트 섀도로 카드를 띄움 |
| 액센트 | 골드 선택강조 | 골드는 절제 → 중립 회색 테두리, 노란기 최소화 |

> 라이트 방향 메모: 백로그 출발점은 "**웜 그레이** 배경"이었으나, 실제 검수에서 크림 톤이 *노랗게* 보여 **뉴트럴하고 더 밝은 그레이**(`#f3f4f6`)로 조정했습니다. "뽀얀·소프트 섀도·흰 카드"라는 핵심 의도는 유지하되, 따뜻함은 과하지 않게.

## 2. Colors

색은 **시맨틱 CSS 변수**로만 씁니다. 컴포넌트에 헥스값을 직접 박지 말 것(아래 Do's & Don'ts 참고).

### 토큰 (원본: `index.html` `:root`)

| 토큰 | 역할 | 다크 (기본) | 라이트 |
|---|---|---|---|
| `--bg` | 페이지 배경 | `#0b0f1a` | `#f3f4f6` |
| `--card` | 카드/패널 면 | `#151b2b` | `#ffffff` |
| `--card2` | 한 단 들어간 면(입력·옵션·순위행) | `#1d2538` | `#f6f7f9` |
| `--line` | 테두리·구분선 | `#2a3450` | `#e6e8ec` |
| `--txt` | 본문 텍스트 | `#e8ecf5` | `#1a1d24` |
| `--sub` | 보조 텍스트 | `#8c97b3` | `#868d9a` |
| `--pill` | 알약 버튼·칩 배경 | `#232c44` | `#eef0f3` |
| `--red` | 한국 반대편/위험/주요 액션 | `#e6394b` | `#e23b4d` |
| `--blue` | 한국팀/활성 탭/보조 액션 | `#3b82f6` | `#2f6bdf` |
| `--gold` | 점수·왕관·강조 텍스트 | `#fbbf24` | `#d4880c` |
| `--green` | 완료·적중·성공 | `#34d399` | `#159b6e` |
| `--sel` | **선택강조 배경 틴트** | `rgba(251,191,36,.12)` | `rgba(30,34,42,.055)` |
| `--selbd` | **선택강조 테두리** | `#fbbf24` | `#aeb4bf` |

`color-scheme`: 다크는 `dark`, 라이트는 `light` (브라우저 기본 컨트롤 동기화).

### 의미 규칙
- **🇰🇷 한국 = 블루, 상대 = 레드** 계열로 좌우 일관.
- **선택 상태**는 `--sel`(배경) + `--selbd`(테두리) **두 토큰으로만** 표현. 다크는 골드로 통일, 라이트는 노란기를 빼고 **중립 회색**으로. → "선택색 바꿔줘"는 이 두 토큰 한 곳만 고치면 전체 반영.
- **골드(`--gold`)는 점수·순위·왕관 등 "값이 빛나야 하는 곳"** 에만. 라이트에선 톤을 낮춰 가독성 우선.
- 성공=그린, 위험·감점=레드, 정보·한국=블루.

### 접근성 · 대비 (M3 참고)
- 텍스트 대비는 **WCAG AA**(본문 4.5:1, 큰·굵은 글자 3:1)를 목표.
- ⚠️ **라이트에서 골드(`--gold`)는 흰 배경 대비가 약함** — 큰·굵은 숫자(점수)에만 쓰고 작은 본문엔 금지. (그래서 라이트 골드를 `#d4880c`로 톤다운함.)
- **색만으로 의미 전달 금지** — 항상 아이콘·텍스트 병행(✅ / 👑 / 🔒 / 취소선 등).

## 3. Typography

서체: `Pretendard` → 폴백 `-apple-system, 'Apple SD Gothic Neo', 'Noto Sans KR', sans-serif`.

타입 스케일은 **Material Design 3의 역할 기반 스케일**(display · headline · title · body · label)을 빌려 *우리 실제 사용 크기*에 매핑했습니다. M3 원본 px를 그대로 쓰지 않고 **역할만 차용** — 모바일 컴팩트 UI라 한 단계씩 작고, 점수 강조를 위해 굵기는 더 무겁게.

| 역할 (M3) | 우리 용도 | 크기 | 굵기 | 줄높이(권장) |
|---|---|---|---|---|
| **Display** | 스코어 (`.mhead .sc`) | 30px | 900 | ~36 (1.2×) |
| **Headline** | 페이지 타이틀 (`h1`) · 순위 점수 (`.pts`) | 20 / 18px | 800 / 900 | ~24 (1.2×) |
| **Title** | 카드 제목 (`.card h3`) | 14px | 800 | ~20 |
| **Body** | 본문 · 칩 · 옵션 | 13px | 600–700 | ~20 (1.5×) |
| **Label** | 배지 · 탭 보조 · 힌트 (`.pt`, `.tab .d`, `.hint`) | 10–12px | 500–800 | ~16 (1.5×) |

- **줄높이 원칙 (M3)**: 큰 글자(title 이상)는 크기의 **1.2배**, 본문·라벨은 **1.5배**. 현재 CSS는 줄높이를 대부분 미지정(브라우저 기본)이라, 다듬을 때 위 권장값을 토큰화하면 리듬이 또렷해짐.
- **굵기 일탈(의도)**: M3 기본은 400/500이지만 우리는 600~900 — 점수·응원의 "또렷함"이 우선이라 의도된 차이.
- **위계 원칙**: 숫자(점수) > 제목 > 본문 > 보조. 점수가 항상 가장 눈에 띄어야 함.

## 4. Layout

- **모바일 우선, 단일 컬럼.** 콘텐츠 최대 폭 **560px** 중앙 정렬(`.wrap`, `.topbar`, 모달 `.mbox`, 토스트).
- **좌우 여백**: `.wrap` `padding:0 12px`.
- **세로 리듬**: 카드 간격 `margin-bottom:14px`, 카드 내부 `padding:20px`(시원하게). 본문 하단 `padding-bottom:84px`(고정 요소·여백 확보).
- **그리드**: 옵션 `.opts.c2`(2열)/`.c3`(3열), 참여자/순위 보조 `.pgrid`(2열), 라이브 버튼 `.lbtns`(2열, `.full`은 한 줄 전체).
- **탭 바**: 가로 스크롤 가능한 `flex`. 경기 탭들 + 종합 탭. **현재 경기 탭은 더 넓게(`flex:1.55`) 강조**, 이후 경기는 흐리게.
- **구분선 여백**: 비교 보드 행(`.brow`)은 `padding:16px 0`으로 위아래 숨 쉬게.

### 스페이싱 스케일 (4 / 8px 그리드, M3 참고)

M3의 **4dp·8dp 배수** 그리드를 기준으로 간격을 정리합니다. 토큰화 제안(원본은 아직 CSS에 px로 흩어져 있음):

| 토큰(제안) | 값 | 배수 | 용도(현재) |
|---|---|---|---|
| `sp-1` | 4px | 4 | 미세 gap · 아이콘 간격 |
| `sp-2` | 8px | 4×2 | 칩·버튼 내부 gap · 기본 간격 |
| `sp-3` | 12px | 4×3 | `.wrap` 좌우 · 카드 내 요소 간격 |
| `sp-4` | 16px | 8×2 | 카드 간격 · 비교행(`.brow`) 상하 |
| `sp-5` | 20px | 4×5 | 카드 내부 `padding` |
| `sp-6` | 24px | 8×3 | 섹션 간 |
| `sp-8` | 32px | 8×4 | 푸터 하단 여백 |
| `sp-10` | 40px | 8×5 | 예고 화면 등 넉넉한 공백 |

> **그리드 이탈값(정리 후보)**: 현재 `6 · 10 · 13 · 14 · 18`px가 4/8 밖. 다듬을 때 `6→8`, `10→8|12`, `13→12`, `14→16`, `18→16|20`으로 수렴 권장. (`padding:20`·하단 `84`는 4-그리드 위 = 유지)

## 5. Elevation

깊이(elevation) 표현 방식이 **테마별로 다릅니다.**

- **다크 = 플랫.** 그림자 거의 없음. **테두리(`--line`)와 배경 단계(`--bg`<`--card`<`--card2`)** 로 면을 구분. 야간 화면에서 그림자는 잘 안 보이므로 톤 차이로 깊이를 줌.
- **라이트 = 소프트 섀도.** "뽀얀" 느낌의 핵심. 흰 카드를 밝은 배경 위에 **부드럽게 띄움**.
  - `.card` : `0 1px 2px rgba(60,50,30,.04), 0 8px 20px rgba(60,50,30,.06)`
  - `.lbrow` : `0 1px 2px …, 0 4px 12px rgba(60,50,30,.05)`
  - `.tab` : `0 1px 2px rgba(60,50,30,.04)` (살짝만)
  - → 그림자는 **따뜻한 회갈색(`60,50,30`) 저투명도**로, 회색이 탁해 보이지 않게.

**Elevation 레벨 (M3식 정리)** — 그림자는 라이트 기준, 다크는 전부 0(톤·테두리로 깊이):

| 레벨 | 적용 | 라이트 그림자 |
|---|---|---|
| 0 | 페이지 배경 · 인풋·옵션 면(`--card2`) | 없음 |
| 1 | 탭(`.tab`) · 칩 | `0 1px 2px rgba(60,50,30,.04)` |
| 2 | 순위 행(`.lbrow`) | 레벨1 + `0 4px 12px …(.05)` |
| 3 | 카드(`.card`) | 레벨1 + `0 8px 20px …(.06)` |
| 오버레이 | 모달 · 토스트 · 데모바 | 별도(오버레이/고정 레이어) |

**레이어(z-index) 순서**: 토스트(99) < 모달(50)… *주의: 데모바(300)가 가장 위.* 모달 배경 오버레이는 라이트에서 `rgba(35,35,45,.42)`로 은은하게.

## 6. Shapes

둥근 모서리로 부드럽고 친근하게. **크기가 클수록 더 둥글게**(radius 스케일).

| 반경 | 적용 |
|---|---|
| 8px | 작은 칩/링크(`.toplink`, 데모바 버튼) |
| 10px | 입력·셀렉트·스테퍼 |
| 12px | 옵션(`.opt`)·참여자 카드(`.pcard`)·순위 행(`.lbrow`)·버튼(`.btn`) |
| 14px | 탭(`.tab`) |
| 18px | 카드(`.card`) — 가장 큰 면, 가장 둥글게 |
| 99px (pill) | 칩(`.chip`)·배지(`.pt`)·토스트 |

- **테두리 두께**: 기본 `1px`, 선택 가능한 요소(옵션·참여자 카드)는 `1.5px`로 또렷하게.
- 아바타는 이모지(원형 느낌), 플래그 이모지로 팀 표현.

## 7. Components

> 상세 구현은 `index.html`. 여기선 "무엇을·언제·어떤 상태로" 쓰는지.

- **Card (`.card`)** — 모든 콘텐츠의 기본 컨테이너. 제목(`h3`) + 우측 배지(`.pt`) + 본문.
- **Button (`.btn`)** — `.p`(레드, 주요 액션) / `.b`(블루, 보조) / `.g`(중립 알약) / `.w`(가로 꽉). 누름 시 `scale(.97)`.
- **Tab (`.tab`)** — 경기/종합 전환. 상태: 기본 / `.on`(선택, 블루) / `.cur`(현재 경기, 강조) / `.soon`(이후 경기, 흐림+🔒).
- **Option selector (`.opt`)** — 예측 보기 선택. 상태: 기본 / `.on`(선택, `--sel`+`--selbd`). 2~3열 그리드.
- **Chip (`.chip`)** — 참여자 목록. `.done`(제출 완료, 그린 테두리+✅).
- **Participant card (`.pcard`)** — 득점자 등 인물 선택. `.on` 선택 강조.
- **Leaderboard row (`.lbrow`)** — 순위 한 줄. `.first`(1위, 👑) / `.open`(펼쳐 항목별 점수) / `.gain`(점수 상승 애니메이션). 순위 변동은 현재 경기 화면(`#lb`)에서만 FLIP 애니메이션.
- **Stepper / Input / Select** — 스코어·시간 입력. `card2` 배경.
- **Badge (`.pt`, `.tag`)** — 점수 배지·메타 라벨.
- **Board chip (`.bchip`)** — 비교 보드. `.uniq`(단독 픽, 레드) / `.hit`(적중, 그린) / `.miss`(빗나감, 흐림+취소선).
- **Toast / Modal / Error bar** — 피드백·다이얼로그·연결 오류.
- **Coming-soon view (`comingSoonView`)** — 아직 안 열린 경기 예고(🔒 + D-day 넛지).
- **Demo bar (`.demobar`)** — `/demo` 전용 상태 전환 패널(운영엔 없음).

**상태 패턴(공통)**: 기본 → 선택(`--sel`/`--selbd`) → 완료(그린) → 적중/빗나감(그린/흐림). 선택은 항상 두 토큰으로만.

### 상호작용 상태 (States, M3 state layer 참고)
M3의 state layer 개념을 가볍게 차용. 현재는 누름 피드백 위주:

| 상태 | 현재 표현 |
|---|---|
| Pressed (누름) | 버튼 `transform:scale(.97)` · 링크 `background:var(--pill)` |
| Selected (선택) | `--sel`(배경) + `--selbd`(테두리) |
| Disabled (비활성) | `opacity:.4` + `cursor:not-allowed` |
| Done / Hit / Miss | 그린 / 그린 틴트 / 흐림 + 취소선 |

> 다듬기 권장: hover/focus 상태를 미세 오버레이로 토큰화(M3 기준 hover 8%·focus 10%·pressed 10%)하면 데스크톱·접근성에서 더 또렷.

## 8. Do's and Don'ts

### ✅ Do
- 색은 **시맨틱 토큰(`--bg`, `--txt`, `--sel` …)으로만** 쓴다.
- 선택 강조는 **`--sel`(배경) + `--selbd`(테두리)** 두 토큰으로 통일.
- 점수·순위 등 **핵심 숫자는 크고 굵게** — 시각적 1순위.
- 라이트는 **밝고 뽀얗게, 노란기는 절제.** 깊이는 소프트 섀도로.
- UI를 고치면 **반드시 `/demo`로 역할×경기상태 전수 확인.**
- 다크/라이트 **둘 다** 점검(토글).
- 이 문서와 `:root`가 어긋나면 **코드를 정답으로 두고** 문서를 갱신.

### ❌ Don't
- 컴포넌트에 **헥스값 직접 하드코딩 금지**(예: `color:#fbbf24`) — 토큰을 거칠 것.
- **`predictions.data`의 jsonb 구조 변경 금지** — 기존 제출과 호환이 깨짐(점수 로직과 직결).
- 새 **라이브러리·프레임워크 추가 지양** — 단일 파일·CDN 한 개 원칙.
- 다크에 **불필요한 그림자** 남발 금지(플랫이 원칙, 톤으로 깊이).
- 라이트에서 **선택 강조를 쨍한 골드로** 쓰지 말 것(노랗게 보임) — 중립 회색.
- 폰트 폴백 체인 임의 변경 금지(한글 깨짐 위험).
- 색만으로 정보 전달 금지 — **아이콘·텍스트 병행**(✅/👑/🔒 등).

---

*초안 끝. 토큰 값은 언제나 `index.html` `:root` 기준 — 이 문서는 그 의도를 적은 지도입니다.*
